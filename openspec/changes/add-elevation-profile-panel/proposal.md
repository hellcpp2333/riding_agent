## Why

骑行路线的核心数据（距离、爬升、下降、海拔剖面）目前只能在对话消息中以文本形式查看，缺乏直观的可视化呈现。地图面板占用全部剩余空间但只展示路线轨迹，空间利用率低。增加海拔剖面面板和爬坡段分析功能，让用户能一目了然地了解路线难度和地形特征。

## What Changes

- 地图容器宽度减少 1/4，为下方新面板腾出空间
- 新增路线数据面板：上方 1/5 显示距离（公里制）、累计爬升（米制）、累计下降（米制），下方 4/5 显示海拔-距离关系图（Canvas 绘制）
- 地图路线上的爬坡段标记：根据 UCI 规则识别坡度段，在地图路线叠加不同颜色标记爬坡段
- 爬坡段侧边栏：从右侧滑出，显示爬坡段序号（第 X 个/共 Y 个）、难度等级、平均坡度、距离、累计爬升，以及该段的海拔-距离图（带坡度颜色编码：绿<3%、黄 3-6%、橙 6-9%、红 9-12%、深红 >12%）
- 高程数据流：后端在路线规划时返回带高程的轨迹点数组（含距离累加值），前端接收后渲染

## Capabilities

### New Capabilities

- `elevation-profile-panel`: 路线数据摘要面板 + 海拔-距离关系图，在页面布局中替换部分地图空间
- `climb-segment-sidebar`: 爬坡段分析侧边栏，支持切换不同爬坡段，显示坡度难度、分段海拔图
- `route-elevation-data`: 后端扩展路线 API 返回带高程和累计距离的轨迹点数组

### Modified Capabilities

_无现有 spec 需要修改。_

## Impact

- `static/index.html` — 布局结构调整（地图容器缩小、新增面板、爬坡侧边栏）
- `static/css/style.css` — 新增面板及侧边栏样式
- `app/api/v1/routes.py` — SSE 事件中附带高程轨迹点数据
- `app/agents/agent.py` — 路线规划后高程数据传递格式调整
- `app/services/elevation_service.py` — 新增轨迹点距离累加计算函数
