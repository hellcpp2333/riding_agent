## Context

当前用户登录后进入"对话"标签页时，消息区域（`#messages`）完全空白。该区域仅在用户发送消息后才有内容（`.msg` 元素动态插入）。对于新用户或新会话，空白区域缺乏引导。需要在不影响现有消息流逻辑的前提下，在空状态时展示功能引导欢迎语。

约束：
- 纯前端改动，无需后端参与
- 需与现有 earth-tone 设计系统保持一致
- 不可破坏现有消息流（SSE）逻辑

## Goals / Non-Goals

**Goals:**
- 消息区域为空时展示欢迎语，简要介绍骑行路线助手的功能
- 用户发送第一条消息后欢迎语自动消失
- 欢迎语视觉上与页面设计协调

**Non-Goals:**
- 不涉及任何后端 API 改动
- 不需要持久化或用户偏好设置
- 不做动画入场效果（保持简洁）

## Decisions

1. **显示/隐藏逻辑：使用 Vue 响应式状态**
   - 通过 `v-if="messages.length === 0"` 在模板中控制欢迎语可见性
   - 无需额外状态变量，直接利用现有 `messages` ref

2. **DOM 位置：放在 `#messages` 容器内部**
   - 欢迎语作为 `#messages` 的子元素，在消息列表为空时展示
   - 当 messages 数组有内容时，消息列表自然替代欢迎语

3. **样式方案：复用现有 `.msg.assistant` 的视觉风格**
   - 欢迎语卡片使用相同的背景色、边框、圆角
   - 采用居中对齐，包含图标（🚴 emoji）、标题、功能列表

4. **不考虑持久化隐藏**
   - 每次刷新页面或新建会话时，欢迎语重新出现
   - 简化实现，无需 localStorage 记录"已展示过"

## Risks / Trade-offs

- [风险] 用户在页面上做其他操作（如切换标签页再切回）时欢迎语重新出现 → **缓解：欢迎语仅在 messages 为空时出现，这是正确的行为——没有对话历史就应该显示引导**
- [风险] 欢迎语内容可能过时 → **缓解：内容为静态的功能介绍，功能不变则介绍无需变更；后续功能扩展时可同步更新文案**
