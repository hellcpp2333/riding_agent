## ADDED Requirements

### Requirement: 欢迎语在空消息栏显示
当对话消息列表为空时，系统 SHALL 在消息区域（`#messages`）展示一段欢迎语，介绍应用的主要功能。

#### Scenario: 新会话显示欢迎语
- **WHEN** 用户进入"对话"标签页且 messages 数组为空
- **THEN** 消息区域显示欢迎语卡片
- **AND** 欢迎语包含应用名称、简短介绍和核心功能列表
- **AND** 欢迎语视觉样式与 earth-tone 设计系统协调一致

#### Scenario: 用户发送消息后欢迎语消失
- **WHEN** 用户发送第一条消息
- **AND** messages 数组不再为空
- **THEN** 欢迎语自动隐藏
- **AND** 正常消息流展示在消息区域

#### Scenario: 切换回空会话时欢迎语重新出现
- **WHEN** 用户切换到另一个 messages 为空的会话
- **THEN** 欢迎语重新展示

### Requirement: 欢迎语内容覆盖核心功能引导
欢迎语 SHALL 至少包含以下功能引导项：
- 路线规划（自然语言描述骑行路线）
- 附近搜寻（搜索周边 POI）
- 路书管理（导入/导出 GPX 文件）

#### Scenario: 欢迎语包含功能引导
- **WHEN** 欢迎语展示
- **THEN** 用户可看到至少 3 项核心功能说明
- **AND** 每项功能配有简短的描述文字
